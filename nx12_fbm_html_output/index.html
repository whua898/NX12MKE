Feature-based machining

Feature-based machining associates a machining feature with a set of instructions to manufacture that feature.

In NX Manufacturing, a machining feature is any shape that is recognized by the software as machinable. Machining features you can select include standard shapes, such as holes, slots, and pockets, and user defined features created for irregularly shaped areas. You can do the following to create machining features:

  • Identify existing modeling features.

  • Recognize features by their parameters if there is geometry that NX cannot identify as a modeling feature.

  • Recognize legacy holes, faces or pockets.

  • Manually recognize features for more control of the feature selection process.

  • Tag geometry or features.

    Tagging adds attributes that feature-based machining needs to create the machining feature. Tagging is required for parts that only contain geometry entities, such as a part imported from a different system. Tagging may be required for features that are missing information such as tolerances or surface finishes.

As you automate the process of machining holes and simple milling features, you can standardize machining these features using best practice machining rules defined within the machining knowledge.

Feature-based machining includes the following:

  • Standardized machining knowledge that defines the valid operations and tools for different features.

  • The Rule-based operations for features technology that helps you automatically create operations for the selected features. Supported operation types include:

    • Milling

    • Turning

    • Wire EDM

    • Drilling

    • Tapping

  • Support for all of the cutting tools available in NX.

  • The Machining Knowledge Editor application for your company's assigned knowledge engineer to customize the supplied machining knowledge.

Note:

The default machining knowledge shipped with NX is built against the cam_general configuration. For the most complete results, start Manufacturing with this configuration.

Feature-based machining workflow

  1. Specify the WORKPIECE geometry.

  2. Open the Machining Feature Navigator and find the machining features with feature recognition or feature identification.

    • Use Identify Features to create machining features from existing modeling features.

    • Use Parametric Feature Recognition if the model does not have modeling features or has additional geometry that is not recognized as a modeling feature.

    The software filters out any faces that are already used for manufacturing features so that each model face will only belong to one feature.

  3. Create geometry groups and operations from the features with the Create Feature Process command.

    The software:

    • Applies the machining knowledge library rules to the selected features and performs any required tests. A rule is only applied in NX if all its conditions are evaluated as TRUE.

    • Adds tools from the tool database.

    • Adds the required operations.

      Note:

      Finishing operations are only added if a feature's finish roughness exceeds the roughness_value attribute value, or its tolerance exceeds the IT grade, specified in the rule library.

    • Displays the results of the reasoning process in the Information window. This includes the order in which rules were applied, and the reasons for rejecting any operations.

  4. Review the suggested operations and tools.

  5. Generate the tool paths.

Feature-based machining support for Wire EDM

You can create feature-based processes for Wire EDM. The standard Feature-based machining installation includes an initial set of predefined Wire EDM features that are recognized, and the Manufacturing Rules for these features.

The set of predefined Wire EDM features includes:

  • WEDM_FREE_SHAPED_STRAIGHT

  • WEDM_OBROUND_STRAIGHT

  • WEDM_RECTANGULAR_STRAIGHT

  • WEDM_ROUND_STRAIGHT

NX automatically creates face boundaries. The boundaries for 2-axis operations are based on a single planar face. The boundaries for 4-axis operations are based on a closed loop of side faces.

Wire EDM start hole data

The Machining Feature Navigator recognizes the following Wire EDM start hole attributes for the set of predefined Wire EDM features:

  • START_HOLE_DIAMETER

    This attribute specifies the diameter of the start hole.

  • START_HOLE_POS_X, START_HOLE_POS_Y

    These attributes specify the relative position of the start hole within the feature. The start hole position is expressed in the coordinate system of the feature.

You must specify a name for the Wire EDM Start Hole Type attribute in the Customer Defaults dialog box. When you create a Wire EDM start hole, the name you specify is assigned as an attribute of the sketch entity that represents the Wire EDM start hole. The default name is PDW_WIRE. The start hole sketch arcs can also be identified independently as manufacturing features.

Machining Feature Navigator

The Machining Feature Navigator palette has two hierarchical views that you use to create and manage all machining features in a Manufacturing part file. The Feature View shows features in the context of the part or assembly to which they belong. The Group View shows features in the context of the workpiece and the MCS. The Machining Feature Navigator also includes the following:

  • An optional customizable Details panel to review and change attributes.

  • An optional Preview panel to view a custom bitmap representing the selected feature and its parameters.

You can change the order and width of the columns in the navigator. To control the feature types and attributes shown, configure and save multiple filters to use on different parts.

You have the ability to view multiple parts at the same time, each in its own graphics window, within the same NX session.

If you expand or collapse the tree structure in the navigator, that structure continues to display the same way if you change to another part, and then return to the first part. The vertical position of the cursor in the tree structure is also saved.

For more information on using multiple windows in NX, see Using multiple windows.

Use the Machining Feature Navigator to:

  • Create feature processes.

    You can control the scope of the feature process by selecting features, or you can create a feature process for the entire part.

  • Find machining features.

  • Create or delete features.

  • Override attributes

  • Create user defined events for each feature and operation.

Create a Machining Feature Navigator filter

  1. In the Machining Feature Navigator , right-click in the background and choose Filter Settings.

  2. In the Filter dialog box, on the Current tab, select an attribute or feature type name.

  3. Select the appropriate criteria from the attribute lists at the bottom of the dialog box. For example, if the selected attribute is Status, select = and Incomplete to specify a filter criteria that only displays features with an Incomplete status.

  4. Click or to add the filter criteria to the Filter Criteria list.

  5. Repeat steps 2–4 for all your filter criteria.

  6. Select the check box in front of each common attribute or feature type that you want to apply in this filter.

  7. Save the filter in one of these ways:

    • Click Save if you want the filter to be available for use in other CAM setups.

    • Click Save in Part if you want to create a part-specific filter. This saves the filter in the file containing your CAM setup.

Apply a Machining Feature Navigator filter

  1. In the Machining Feature Navigator , right-click in the background and choose Use Filter.

  2. Select a filter from the list.

You can also apply a filter from the Filter dialog box:

  1. In the Machining Feature Navigator , right-click in the background and choose Filter Settings.

  2. Click the Saved tab to access filters saved in the system registry, or click the Saved in Part tab to access filters saved with the CAM setup file.

  3. Select a filter from the list.

  4. Click Apply .

Filter by the MCS tool axis

To show only features that are parallel to the tool axis defined in the MCS:

  1. In the Machining Feature Navigator , right-click in the background and choose MCS Scope.

  2. Select an MCS from the list. Every MCS with a tool axis defined is shown in the list.

The Machining Feature Navigator displays only features that are parallel to the tool axis of the selected MCS, and the Name column displays the name of the selected MCS.

Show only Hole Making operations

  1. In the Operation Navigator , right-click in the background and choose Create Filter.

  2. In the Operation Filter dialog box, from the Type list, select Hole Making.

  3. Click OK.

Feature update acceptance criteria

When the part geometry changes, the status for a feature group changes to Suspected if any of the following criteria are not matched in one or more features in the group:

  • Feature type

  • Parameter classification

  • Feature tool axis

  • Part geometry

  • Processor tool axis limitations.

  • Processor geometry limitations. For example, Milling boundary geometry cannot define equal features with different floor definitions.

Updates that cause a mismatch between the feature and the parent feature group include:

  • Changes to the face list of a 加工 feature. For example, deleting a face.

  • Changes to the feature parameters. For example, changing a face that determines the length or width of a 加工 feature.

  • Changes to a 建模 feature that affect 加工 feature parameters. For example, changing the diameter of an advanced hole feature.

  • Redefining the part geometry of the Workpiece.

  • Redefining the tool axis parameters of the MCS.

  • Redefining the classification criteria of the feature group.

You can take the following corrective actions:

  • Manually remove the feature that is causing the problem.

  • Modify the feature group.

    You can modify the feature group classification criteria, or drag the feature group to a new parent.

  • Use the Redistribute command to restore the feature geometry distribution of the selected feature group.

  • Use the Recreate command to create a new feature process for the selected feature group.

STEP2HOLE

Determining if step holes require pre-machining

Feature-based Machining uses the TILTED_TOP_DEPTH and the TILTED_BOTTOM_DEPTH attributes in conditions to determine whether a hole can be drilled directly, or requires a milling operation to pre-machine the hole.

  • STEPPOCKET features include the TILTED_TOP_DEPTH (1) attribute.

  • STEPHOLE features include the TILTED_TOP_DEPTH and the TILTED_BOTTOM_DEPTH (2) attributes.

During automatic feature recognition, the TILTED_TOP_DEPTH attribute replaces the pre-NX 9.0.2 attribute TOP_FACE_ANGLE. When you open a pre-NX 9.0.2 part that already contains recognized stepped features, NX:

  • Adds the TILTED_TOP_DEPTH and TILTED_BOTTOM_DEPTH attributes.

  • Calculates the TILTED_TOP_DEPTH value based on the existing TOP_FACE_ANGLE attribute and the feature diameter.

  • Sets the TILTED_BOTTOM_DEPTH attribute to the default value of 0.0.


Because NX does not perform an update recognition until you make a design change, the TILTED_TOP_DEPTH and TILTED_BOTTOM_DEPTH values may not be accurate. Design changes to a part force an update of the feature recognition. During the update of a pre-NX 9.0.2 part, NX:

  • Evaluates the geometry and assigns accurate values to the TILTED_TOP_DEPTH and TILTED_BOTTOM_DEPTH attributes.

  • Marks the feature as Changed and Updated if either of the following apply:

    • The newly recognized TILTED_TOP_DEPTH value does not match the former TOP_FACE_ANGLE value.

    • The TILTED_TOP_DEPTH value is not the default value of 0.0.

Using Attributes For Classification

After the Holemaking module has identified the features in a part according to their CAM_FEATURE_NAME attribute, the system then groups the features together based on their other attribute values such as diameter and tolerance.

Attributes attached to geometric entities or features, can be used as classification criteria to classify the features based on these attributes. Therefore, features are grouped together into feature groups depending on the chosen classification criteria.

Classification attributes are given the same names as base feature parameters in order to allow manufacturing parts to contain mixed feature types. Using the same names also keeps the Holemaking template from getting too complex.

Amount of Attribute Information Needed Per Entity Tag

Some entities need more attributes than others depending on the tagged entity (face, edge, arc, point). For example, a tagged face does not necessarily need a depth attribute because the Holemaking template can use the Model Depth for the depth calculation. However, on a point a depth attribute is essential because there is no Model Depth to draw from. Every tagged entity needs to have certain attributes to make it a valid candidate for the Holemaking process.

Below is an example of how the different entity tags need different amounts of information for the same CAM_FEATURE_NAME value.

Minimum attribute requirements for a SIMPLE_HOLE:

Face: CAM_FEATURE_NAME, DIAMETER, I, J, K

Edge: CAM_FEATURE_NAME, DIAMETER

Arc: CAM_FEATURE_NAME, DIAMETER, DEPTH, I, J, K

Point: CAM_FEATURE_NAME, DIAMETER, DEPTH, I, J, K

Using Minimum Attributes

You should always apply the same set of attributes to all entities on a mixed feature type model. This is because only the minimum set are taken into account for feature classification.

Example

A model contains a tagged point and a base feature. The tagged point is tagged with SIMPLE_HOLE as its CAM_FEATURE_NAME. It also has an additional attribute, TOLERANCE. The base feature is also a SIMPLE_HOLE, with the same parameter values as the tagged point (diameter, depth, I,J,K). However, it does not have the addition of the TOLERANCE attribute.

Holemaking tries to group these entities based on the same characteristics. The TOLERANCE attribute does not display in the Classification Criteria field in Holemaking because it is not an attribute that both tagged entities contain. User Defined Attributes (UDAs)

Rule

You must define User Defined Attributes (UDAs) on features, faces or edges either at the component level or at the occurrence at the assembly level. Attributes in subassemblies are not considered. Mixed tagging is allowed as long as they share the same geometry entity. Notice that UDAs defined at the assembly level always take precedence over those defined at the component level. This allows a simple modification of the original value without touching the master model.

Below are two examples:

Example 1: If a feature in the component has an attribute named TOLERANCE with a certain value you can overwrite this by defining a UDA named TOLERANCE on the feature, face or edge at the assembly level.

Example 2: You can disable the feature COUNTER_BORE_HOLE in the component by tagging the UDA CAM_FEATURE_NAME with the value SIMPLE_HOLE on the feature, face or edge at the assembly or component level. But by doing this you wipe out all COUNTER_BORE_HOLE parameters, and they are no longer used.

Other Attributes for Classification

Standard UDF (Non KF enabled): Classification of all input parameters + additional UDAs on feature, face or edge.

KF enabled UDF with KF attributes: Classification upon all KF attributes, additional UDA considered only in the occurrence of the assembly.

Note:

You must define all additional KF attributes directly in KF. You must define all additional UDAs on the faces of the UDFs in the assembly.

KF enabled UDF without KF Attributes: Classification upon all input parameters + additional UDA on feature, face or edge.

Note:

Once you define a KF attribute the input parameters are no longer considered.

Creating geometry groups and operations

Use the Create Feature Process command to create geometry groups and operations for features. NX:

  • Applies the machining knowledge library rules to the selected features and performs any required tests. A rule is only applied in NX if all its conditions are evaluated as TRUE.

  • Adds tools from the tool database.

    To reduce the number of tools and tool changes, NX first tries to select library tools that are already used in the part file. If tools that match the requirements are not available in the part file, then NX retrieves additional tools from the library.

  • Adds the required operations.

    Note:

    Finishing operations are only added if a feature's finish roughness exceeds the roughness_value attribute value, or its tolerance exceeds the IT grade, specified in the rule library.

  • Displays the results of the reasoning process in the Information window. This includes the order in which rules were applied, and the reasons for rejecting any operations.

To fully automate the process, use the Rule Based option. NX creates operation sets for you based on the rules in the machining knowledge library. If your company has created operation sets to establish your best practices, use the Operation Set Based option. NX filters the operation sets based on their application conditions and displays only the operation sets that you can use for your feature type. You then select the set that is best for your situation.

Optimization

Optimization improves cutting efficiency by applying rules to the NC program. Optimization uses machining feature templates to give the NC programmer the most efficient operations to machine each feature and to optimize the order in which they are performed.

Operations are optimized by the conservation of tools, minimizing tool changes, and minimizing tool travel distance (the amount of time the tool is in the air moving from one cut to another).

  • Optimization conserves tools by using as few tools as possible without compromising the effectiveness of machining across multiple operations. These tools are selected from the result of the tool query of each operation, which returns a set of tools from the library.

  • To minimize tool changes, operations that use the same tool are grouped together.

  • In order to reduce the tool travel distance the system selects the best sequence in which to cut the features.

All three of these optimization acts save time during machining. However, minimizing the amount of tools that are used also reduces tool cost, and improves machining efficiency. These are three methods of optimization that can be performed to get the most efficient program.

To optimize the order of the operations in the Program View, select either an NC program, or a single program group. Below are a few guidelines regarding Program View optimization.

  • Selected operations in consecutive order within the same NC program are reordered together.

  • If selected operations are not in consecutive order, the reordering scope is among the consecutive operations only.

  • Operations selected from different NC program subgroups are reordered separately.

  • If the NC program contains a sub NC program, this optimization is applied to the sub program separately without breaking up the sub NC program.

Teach operation sets - standardized process

The following example shows the recommended workflow that companies should use to teach NX the set of operations required to machine company feature types. The example supposes that your company wants to teach NX operation sets to standardize the way you machine holes. You can use the same workflow to teach NX any feature type. This example uses a separate teach part, which is preferred, but you can use any part to teach operation sets.

Suppose that your company wants to automate the following requirements:

  • Recognize holes that must be machined to tolerance separately from holes that do not require a specific tolerance, and assign them different names. Standard holes are any color except yellow. Toleranced holes have one or more of the following characteristics:

    • A yellow color

    • A PMI dimension tolerance grade ≤ IT 7

    • An assigned face attribute

  • Drill a standard hole to size if the matching drill bit is available. Otherwise, drill the hole and mill it to size.

  • Drill and bore a toleranced hole if IT grade > 5.

  • Drill and bore a toleranced hole and then ream it to size if IT grade ≤ 5.

In this workflow you will:

  1. Teach NX to map the standard feature types to your company's feature types.

    Note:

    This step is not necessary if you do not want custom names for your features.

  2. Create the feature groups and operations you need to teach.

    This example includes 4 feature groups and the operation set required to machine each group. All of the operation sets include a countersink. In the testing step, you will see that NX omits this operation when it is not needed.

  3. Teach the operation sets.

  4. Test the operation sets teaching.

    You will use the Rule Based option of the Create Feature Process command, because this workflow is for an automated process where NX decides which operation set to use for the selected features.

Note:

  • You must have write permissions for your machining knowledge library, and a Machining Knowledge Editor Author license to teach the feature mappings and operation sets. We recommend that you make a backup copy of your machining knowledge library before you try this workflow.

  • The parts used in this workflow are available in the mach/samples folder of your NX installation.

Feature teaching requirements and error conditions

The following requirements must be met for you to teach NX a custom feature:

  • You must have write permissions to the machining knowledge library.

  • The faces you select to define the feature must be connected.

  • The feature name must be unique.

  • The recognition rule name must be unique.

NX displays an error message if, for example, you attempt to give a new feature a duplicate name.

The feature should also have a coordinate system to position and orient the feature accurately.

Note:

  • The coordinate system must be a Datum CSYS.

  • NX does not require a coordinate system to teach the feature.

Positioning warning conditions

NX ignores a CSYS that does not reference any of the feature geometry. Only the first valid CSYS object is used. If additional valid CSYS objects exist, the following message is displayed.

WARNING: ‘CSYS name’ is skipped: parameter ‘POSITION’: already defined

If the feature you teach NX references a point, edge, or a face that does not belong to the feature geometry, the following message is displayed.

WARNING: ‘CSYS name’ is skipped: unrelated reference geometry

The Point on Face and At Angle to XC options are not supported. If the feature you teach NX uses an unsupported point or vector type in one of its geometry references, the following message is displayed.

WARNING: ‘CSYS name’ is skipped: unsupported auxiliary geometry type (reference description)

If the feature you teach NX is symmetric and would be recognized with different positions and/or orientations, the following message is displayed.

WARNING: ‘Rule name’ taught feature has alternative directions (#)

The Teach Features command will still teach NX the feature type and store the recognition rule in the recognition library. However, NX will not have the information that is required to position and orient the feature accurately.

  • Solution: Replace the CSYS with a CSYS that is supported for feature teaching.

    Feature teaching fully supports all CSYS subtypes which reference a combination of points, edges, and faces or datums of the feature geometry.

In Process Workpiece

The In Process Workpiece option has two settings: None and Use 3D.

When you select None, the default option, the system doesn't use an in process workpiece.

When you select Use 3D, the system adjusts the cycle RAPTO (clearance distance) and RTRCTO (retract distance) according to the current IPW. Also, if you select Collision Check and Use 3D for an operation, the collision check system references the current IPW for possible collision during transition moves.

During the vertical transition move, the move from a point high above the hole to the hole top, the collision is determined as follows:

  • If the tool or the holder collides with part or check geometry, then it is a collision.

  • If the holder collides with the IPW, then it is a collision.

All the IPW rules defined for Cavity Mill apply to Holemaking.

Collision Check

When selected, the software checks for collisions during the transition and approach moves and avoids the collisions. The software uses the following clearance related options when Collision Check is selected: Safe Clearance, Clearance Distance and Vertical Clearance.

A collision is any movement where the tool tip comes within the axial or radial Safe Clearance distance around the part. The tool tip moves to either the Clearance Distance or Vertical Clearance level to avoid the collision.

If there is a tool axis change between two holes, the operation does not perform a collision check. Instead, the tool retracts from the previous hole to the distance specified for the Added Retract on Toolaxis Change option before transitioning to the next hole.

When checking for collisions the system checks indefinitely in the direction of the hole axis for any collision during transition or approach. However, if a tool is short enough to traverse under an overhang condition and clear the part geometry then no collision is detected. If there is part geometry along the hole axis, the system reads this as a collision no matter how far the part geometry is from the feature being machined. Be aware of this when planning your program so that your program doesn't result in unmachined features.

If Collision Check is turned on with the Hole Depth Type set to Rule, and the Depth value is larger than the tool length, the hole will not be machined.

See also: Set the IPW precision for collision checking

Cut Method

Cut types in hole mill operations determine the tool path pattern used to machine cut regions. The following cut types are available for Hole Mill operations: spiral and helical.

Spiral

The spiral cut pattern cuts one level at a time in a spiral pattern beginning in the center of the hole, and ending when it has reached the wall. Then it makes one more circular pass along the wall of the hole.

When a predrill diameter is defined to be larger than the tool diameter, the spiral cut pattern begins at a point on the wall of the predrilled hole.

  1. The green circle is the edge of the hole you are cutting.

  2. The blue outer circle is the wall clean up cutting move.

  3. The red line is the rapid move.

  4. The yellow curve is the engage move.

  5. The dotted white line is the retract move at rapid speed.

  6. The blue arc is the retract move at the cutting speed.

  7. The blue spiral is the cutting move.

Helical

The helical cut pattern goes along the wall of the hole in a continually downward motion, until it reaches the bottom of the hole or the bottom of the level. When it reaches the bottom of the level it makes one more circular pass along the wall of the hole.

  1. The yellow line is the engage move.

  2. The solid green line is the part.

  3. The dotted green line is the part.

  4. The blue helix is the cutting move.

  5. The bottom blue circle is the bottom clean up cutting move.

Ramp Type

Ramp type enables you to specify how the tool ramps into the material when engaging. The Ramp Type drop down menu contains the following three options: On Line, On Shape and Helical.

On Line

Selecting On Line allows ramping to occur only along linear cuts.

On Shape

Selecting On Shape allows ramping to occur along all traced cutting passes regardless of shape.

The helical ramp type is made with a 10% overlap of the tool to prevent a post from remaining in the middle of the helix. It is only performed if it does not gouge the part while engaging.

Ramp Angle (KF)

Ramp angle is the angle at which the tool cuts into the material while performing on lines, on shape, or helical ramping. It is measured in a plane perpendicular to the part surface. The ramp angle determines the starting location of the tool, since the tool must be near the starting position of the first cut when it finishes descending to the cutting level. In the Ramp Angle field, specify any value greater than 0 degrees but less than 90 degrees. The tool starts the ramping move at the point of intersection between the ramping angle and the specified vertical clearance. No ramping occurs if the area to be cut is smaller than the radius of the tool.

When Ramp Type is set to Helical, the Helical Diameter % option becomes available. Helical diameter % specifies the desired or maximum ramp diameter for a helical engage.

The Minimum Ramp Length-Diameter % option specifies the minimum ramp length or diameter for on lines, on shape, or helical engage. You should set the Minimum Ramp Length-Diameter whenever you perform a ramping or helical engage into material with a non-center cutting tool, such as an insert cutter . This ensures that a small island or pillar of material under the center of the cutter is not left uncut by a ramping engage motion.

Warning:

Subsequent use of the From Point establishes a new reference position without movement. This may produce an inaccurate part because the next set of output points will have a different reference than the previous set of output points.

Tool Axis uses the Vector dialog box to specify the tool axis orientation. If you do not specify a tool axis, the software automatically assigns the default 0,0,1 tool axis value of the Machining Coordinate System (MCS).

Start Point

Specifies a tool position that avoids geometry and fixture components prior to the start up sequence.

The cutter will move at the Rapid feed rate as follows:

  • If a clearance plane has not been defined, to the Start Point.

  • If a clearance plane has been defined, to a point on the Clearance Plane directly above the Start Point, and then to the Start Point.

The cutter then moves at the approach feed rate as follows:

  • If an engage move has not been defined, directly to the first point on the cutter path.

  • If an engage move has been defined, to the beginning of the engage move.

Return Point

Specifies a tool position away from the part at the end of a cutting sequence.

The cutter moves at the return feed rate from the end of the retract move, if defined, or from the last point of the cutter path, as follows:

  • If a clearance plane has not been defined, directly to the Return Point.

  • If a clearance plane has been defined, straight up to the clearance plane, and then to the Return Point. The Return Point should be located on or above the clearance plane.

Tool Axis specifies the tool orientation.

Retain Last

Specifies the same tool axis as the last GOTO point.

MCS (0,0,1)

Aligns the tool axis with the Z-axis of the MCS.

Go Home Point

Specifies the final tool position.

Use From Point positions the Go Home Point in the same location as the From Point. The symbol FR:GH displays in the graphics window at the From Point location to indicate that the From Point and Go Home Point are at the same location.

Gohome with No Parameters outputs a move to the GOHOME position without location values or tool axis.

Clearance Plane

Defines a safe clearance distance for tool motion before and after an operation and during any programmed obstacle avoidance. You can specify a clearance plane that is associative or non-associative.

Use At

Specifies where to use the clearance plane.

Start and End

Generates a move to the clearance plane at both the start and end of the tool path.

Start Only

Generates a move to the clearance plane at the start of the tool path only. At the end of the path, the tool retracts in one of the following ways:

  • For Hole Making operations, the tool retracts by the specified Retract Distance value.

  • For Hole Milling operations, the tool retracts by the specified retract method. If a retract move is not specified, the tool remains on the part surface.

End Only

Generates a move to the clearance plane at the end of the tool path only.

For Hole Milling operations, the tool engages using the specified engage method. If an engage method is not specified, the first motion is the first cut motion on the part surface.

Start Only, Min Clear at End

Generates a move to the clearance plane at the beginning of the tool path. At the end, it retracts along the tool axis a distance equal to the specified Minimum Clearance or Retract Distance value.

End Only, Min Clear at Start

Generates a move to the clearance plane at the end of the tool path. The tool path begins with an engage move along the tool axis a distance equal to the Minimum Clearance or Retract Distance value.

Lower Limit Plane

Defines the lower limit of cutting and non cutting tool motions. The normal vector defines the above side of the plane and should point toward the tool holder.

Action

Specifies what to do when the lower limit plane is violated.

Warning Only

The GOTOs are not affected. The tool motion is not altered and is allowed to violate the Lower Limit Plane, but a warning is written to the tool path and CLSF.

Normal to Plane

Displays a warning and projects the violating GOTOs to the lower limit plane, in a direction normal to the lower limit plane. Omits the violating tool motion.

Along Tool Axis

Displays a warning and projects the violating GOTOs to the lower limit plane, in a direction along their respective tool axis. Omits the violating tool motion.

Limit Point

Available for Drill operations.

Omits points below the lower limit plane. For points on the lower limit plane where FEDTO causes a violation, adjusts FEDTO to reach the lower limit plane.

Flip Normal to Plane

Reverses the normal vector of the defined lower limit plane.

Note:

The cutter may gouge the lower limit plane when you use either projection method and the tool axis is not normal to the lower limit plane.

Redisplay Avoidance Geometry

Avoidance Geometry common options

Specifies the point location using the Point Constructor. The avoidance geometry symbol appears on the display screen at the specified location and remains displayed until the screen is refreshed. The avoidance geometry status changes from None to Active.
