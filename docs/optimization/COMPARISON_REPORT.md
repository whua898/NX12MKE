### 常见问题与解决方案

| 问题 | 可能原因 | 解决方案 |
|------|---------|---------|
| 规则不命中 | 窗口设置过窄 | 放宽Application Criteria窗口 |
| 双命中冲突 | 优先级设计不合理 | 调整Priority或增加互斥条件 |

| 操作参数未生效 | 在Conditions中赋值复杂参数 | 迁移至Add-On附加工艺模板 |

**排查步骤**：
1. 保产线：启用 `ManualCheck` 占位，不让特征漏加工
2. 查语法与字段：确认模板分段、字段名可用
3. 查窗口与优先级：先放宽命中，再回收精度
4. 留痕：更新 Change Log，记录 Owner/Date/Delta

**优势**：
- ✅ 列出5类常见问题

### 自动化文档生成

**目标**：从JSON/XML自动生成Markdown表格

**实现思路**：
```python
# 伪代码
def generate_rule_table(rules_json):
    """从JSON生成规则表格"""
    table = "| 规则名 | Priority | 应用条件 |\n"
    table += "|--------|---------|----------|\n"
    for rule in rules_json:
        table += f"| {rule.name} | {rule.priority} | {rule.criteria} |\n"
    return table
```

**收益**：
- ✅ 减少人工维护成本
- ✅ 保证数据一致性
- ✅ 实时更新

### 规则冲突检测脚本

**目标**：自动识别同路径同优先级规则

**实现思路**：
```python
# 伪代码
def detect_conflicts(rules_xml):
    """检测规则冲突"""
    conflicts = []
    for path in get_all_paths(rules_xml):
        rules = get_rules_by_path(path)
        priority_groups = group_by_priority(rules)
        for priority, group in priority_groups.items():
            if len(group) > 1:
                conflicts.append({
                    'path': path,
                    'priority': priority,
                    'rules': group
                })
    return conflicts
```

**收益**：
- ✅ 提前发现潜在冲突
- ✅ 减少人工审查时间
